@startuml Диаграмма последовательности
title Просмотр расписания врача

actor "Администратор" as Admin
participant "Веб-интерфейс\n(страница расписания)" as UI
participant "Сервис расписания" as Service
database "База данных" as DB

Admin -> UI: Открывает страницу расписания
UI -> Admin: Отображает фильтры\n(дата, врач, услуга)

Admin -> UI: Выбирает врача и дату
UI -> Service: Запрос расписания

Service -> DB: Получить стандартное расписание врача
DB --> Service: Рабочие интервалы\n(например, среда 09:00-18:00)

Service -> DB: Получить исключения по врачу и дате
DB --> Service: Исключения (например,\n 01.03.2026 - 07.03.2026 отпуск)

Service -> DB: Получить записи врача на дату
DB --> Service: Существующие записи

Service -> Service: Сопоставить стандартное расписание,\nисключения и существующие записи
Service -> Service: Вычислить доступные, занятые\nи недоступные интервалы
Service -> Service: Сформировать модель сетки\nрасписания в виде JSON

Service --> UI: JSON с данными расписания

UI -> UI: Построить временную сетку\nпо интервалам из JSON

UI --> Admin: Показать готовую сетку расписания

@enduml
