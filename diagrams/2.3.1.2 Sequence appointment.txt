@startuml Диаграмма последовательности
title Создание записи на прием

actor "Администратор" as Admin
participant "Веб-интерфейс\n(страница записи)" as UI
participant "Сервис записей" as Service
database "База данных" as DB

Admin -> UI: Открывает страницу создания записи
UI -> Admin: Отображает форму записи\n(пациент, врач, дата, время, услуга)

Admin -> UI: Заполняет данные записи\n(выбирает пациента, врача, слот, услугу)
UI -> Service: Запрос создания записи

Service -> DB: Получить данные пациента
DB --> Service: Данные пациента\n(ФИО, контакты)

Service -> DB: Получить данные услуги
DB --> Service: Данные услуги\n(название, длительность, цена)

Service -> DB: Получить расписание врача на дату
DB --> Service: Стандартное расписание,\nисключения, существующие записи

Service -> Service: Проверить доступность выбранного слота\n(учитывая расписание, исключения, записи)

alt Слот свободен
    Service -> DB: Создать новую запись\n(пациент, врач, время, услуга, статус "создана")
    DB --> Service: ID новой записи
    
    Service -> DB: Записать в логи\n(создана запись администратором)
    DB --> Service: Логи записаны
    
    Service -> Service: Сформировать ответ\nс данными новой записи
    Service --> UI: Успешно создана запись\n(данные записи)
    
    UI -> UI: Обновить сетку расписания\n(пометить слот как занятый)
    UI -> Admin: Показать подтверждение создания\nс деталями записи
else Слот занят или недоступен
    Service -> DB: Записать в логи (ошибка\nсоздания записи администратором)
    DB --> Service: Логи записаны
    Service --> UI: Ошибка - слот недоступен\n(причина: 'указать конфликтующий элемент')
    
    UI -> UI: Показать ошибку валидации
    UI -> Admin: Показать сообщение об ошибке\nи подсветить недоступный слот
end

@enduml
